2. MC/DC 설계 도구

MC/DC(Modified Condition / Decision Coverage)의 개념은 다음과 같다

  • 개별 조건식이 다른 조건식의 영향을 받지 않고 전체 조건식의 결과에 독립적으로 영향을 주는 구조 기반 테스트 설계 기법
  • 개별 조건식이 전체 조건식의 결과에 영향을 주는 조건 조합을 찾아 커버리지를 테스트하는 방법

 

MC/DC의 기본 사상은 다음과 같다. 

if (A && B){
    statement;
}

의 전체 케이스는

 

A와 B를 테스트하는 모든 케이스는 22 = 4 가지이다. 만일 if ( A && B && C) 의 경우 모든 테스트 케이스는 23 = 8가지가 될 것이다.

 

이를 다중 조건 조합 테스트(Multiful Condition Coverage) 라고 하는데 이의 장점과 단점은

  • 장점 : 완벽한 테스트 (일어날 수 있는 모든 케이스)
  • 단점 : 너무 수행해야 할 케이스가 많다.

이다.

 

이를 효율적으로 줄이기 위해 나온 방법이 변형된 조건/결정 컨디션(MC/DC) 방법이며 세부적인 방법은 여기에서 설명을 하지는 않겠다. 이 기법이 현재 테스팅에서 차지하는 위치는 확고 부동이며 사람의 생명 및 재산 손실이 달려 있는 자동차, 철도 및 항공 우주 시스템에 대해서는 구조 기반 테스트 시 이 기법의 활용이 절대적이라 할 수 있다. 더 자세한 내용은 구글링을 참조하기 바란다.

여기에서 제공하는 MC/DC 설계 도구는 Pseudo Code의 조건절마다 MC/DC를 구한 다음에 각 조건절의 MC/DC결과들의 조합을 케이스로 만든다. (이를 MC/DC를 활용한 요소 비교 테스팅(Elementary Comparison Testing)이라고 한다) 예제를 보자.
